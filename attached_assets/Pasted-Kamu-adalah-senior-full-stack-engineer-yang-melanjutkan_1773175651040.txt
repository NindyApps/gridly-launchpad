OCTOPILOT memantau Reddit & Hacker News secara real-time untuk menemukan 
postingan yang menunjukkan sinyal beli (misalnya "looking to switch from 
Salesforce", "anyone recommend a CRM?"). Sinyal diklasifikasi dengan AI 
(GPT-4o-mini), lalu didorong ke HubSpot CRM sebagai task follow-up.

Target user: Tim sales & marketing B2B SaaS.

LANGKAH 1 — Signal Ingestion Engine
(PRIORITAS TERTINGGI — tanpa ini produk tidak berguna)
═══════════════════════════════════════

Tujuan: Membuat sistem yang secara otomatis mengambil postingan dari 
Reddit dan Hacker News, mencocokkan dengan tracker aktif, 
mengklasifikasi dengan OpenAI, dan menyimpan ke DB.

1a. Buat file src/lib/scrapers/reddit.ts
    - Fungsi: fetchRedditPosts(subreddit: string, keywords: string[])
    - Gunakan Reddit JSON API: https://www.reddit.com/r/{sub}/new.json
    - Filter postingan yang mengandung salah satu keyword di title/body
    - Return array posts dengan field: id, title, body, url, author, 
      created_at, subreddit, score
    - Handle rate limit dengan exponential backoff
    - JANGAN gunakan library scraping berat — cukup fetch + JSON

1b. Buat file src/lib/scrapers/hackernews.ts
    - Fungsi: fetchHNPosts(keywords: string[])
    - Gunakan Algolia HN Search API: 
      https://hn.algolia.com/api/v1/search_by_date
    - Query parameter: query="{keyword}", tags=story, 
      numericFilters=created_at_i>{unix_24h_ago}
    - Return format sama seperti reddit.ts
    - Handle pagination (hitsPerPage: 20)

1c. Buat file src/lib/ingest.ts
    - Fungsi utama: runIngestionForTracker(trackerId: string)
    - Alur:
      1. Ambil tracker dari DB berdasarkan trackerId (include keywords, 
         competitor_names, subreddits, platform)
      2. Jalankan scraper sesuai platform tracker (reddit/hn/keduanya)
      3. Untuk setiap post yang ditemukan, cek apakah sudah ada di tabel 
         intent_signals berdasarkan external_id (hindari duplikat)
      4. Kirim post ke fungsi classifySignal() di src/lib/openai.ts
      5. Simpan hasil ke tabel intent_signals di Supabase
    - Fungsi pendukung: runIngestionForAllTrackers()
      → Ambil semua tracker aktif (is_active = true) dari semua workspace
      → Jalankan runIngestionForTracker() untuk masing-masing
      → Log hasil ke console: berapa post ditemukan, berapa disimpan

1d. Buat API route src/app/api/ingest/run/route.ts
    - Method: POST
    - Auth: Cek header Authorization dengan nilai dari env CRON_SECRET
    - Body (optional): { trackerId?: string }
    - Jika trackerId ada → jalankan untuk satu tracker
    - Jika tidak ada → jalankan untuk semua tracker aktif
    - Response: { success: true, processed: number, saved: number }
    - Tambahkan CRON_SECRET ke .env.example

1e. Buat Supabase Edge Function (opsional tapi direkomendasikan)
    - File: supabase/functions/ingest-signals/index.ts
    - Schedule: setiap 15 menit via pg_cron atau Supabase cron
    - Isi: POST ke /api/ingest/run dengan header CRON_SECRET
    - Alternatif jika Edge Function tidak tersedia: dokumentasikan cara 
      setup cron job eksternal (misalnya via cron-job.org) yang hit 
      endpoint /api/ingest/run

VERIFIKASI LANGKAH 1:
- `npm run dev` → tidak ada TypeScript error
- Hit POST /api/ingest/run dengan Postman/curl + header Authorization yang benar
- Cek tabel intent_signals di Supabase → harus ada data baru masuk
- Buka /dashboard → signal feed harus menampilkan sinyal nyata (bukan seed)
- `npm run build` → sukses tanpa error

CHECKPOINT 1: Jika semua verifikasi lewat, commit/catat sebagai 
"feat: signal ingestion engine"

LANGKAH 3 — HubSpot Token Refresh
═══════════════════════════════════════

Tujuan: Token OAuth HubSpot kedaluwarsa ~30 menit. 
Implementasi auto-refresh agar inject ke CRM tidak gagal.

3a. Buat fungsi src/lib/hubspot.ts (extend file yang sudah ada)
    - Tambahkan fungsi: refreshHubSpotToken(workspaceId: string)
    - Logic:
      1. Ambil hubspot_refresh_token_enc dari tabel workspaces
      2. Dekripsi token (gunakan metode enkripsi yang sudah ada di project)
      3. POST ke https://api.hubapi.com/oauth/v1/token dengan:
         grant_type: 'refresh_token'
         client_id: HUBSPOT_CLIENT_ID
         client_secret: HUBSPOT_CLIENT_SECRET
         redirect_uri: HUBSPOT_REDIRECT_URI
         refresh_token: decryptedRefreshToken
      4. Enkripsi access_token dan refresh_token baru
      5. Update workspaces: hubspot_access_token_enc, 
         hubspot_refresh_token_enc, hubspot_token_expires_at

3b. Buat fungsi wrapper: getValidHubSpotToken(workspaceId: string)
    - Cek hubspot_token_expires_at di DB
    - Jika expires dalam < 5 menit → jalankan refreshHubSpotToken() dulu
    - Return access token yang valid
    - Gunakan fungsi ini di semua tempat yang butuh HubSpot token

3c. Update src/app/api/crm/hubspot/inject/route.ts
    - Ganti pengambilan token langsung dari DB dengan 
      getValidHubSpotToken(workspaceId)
    - Tangani error jika refresh gagal (tampilkan pesan yang jelas ke user)

3d. Tambahkan kolom hubspot_token_expires_at ke tabel workspaces 
    jika belum ada:
    - Buat file migrasi SQL: supabase/migrations/add_hubspot_token_expiry.sql
    - Isi: ALTER TABLE workspaces ADD COLUMN IF NOT EXISTS 
      hubspot_token_expires_at TIMESTAMPTZ;
    - Update callback OAuth di src/app/api/crm/hubspot/callback/route.ts:
      → Hitung dan simpan expires_at saat token pertama kali diterima
      → expires_at = now() + expires_in detik (dari response HubSpot)

VERIFIKASI LANGKAH 3:
- `npm run dev` → tidak ada error
- Connect HubSpot, tunggu 35 menit (atau manual set expires_at ke masa lalu 
  di DB untuk simulasi), lalu coba inject signal
- Inject harus tetap sukses (tidak gagal karena expired token)
- Cek console log → harus ada log "HubSpot token refreshed for workspace X"
- `npm run build` → sukses

CHECKPOINT 3: "feat: hubspot token auto-refresh"

LANGKAH 4 — Alert Notification Delivery
═══════════════════════════════════════

Tujuan: Saat sinyal baru dengan confidence >= threshold tersimpan ke DB,
kirim notifikasi ke email (via Resend) dan/atau Slack webhook.

4a. Buat fungsi src/lib/notifications.ts
    - Fungsi: sendAlertNotification(signal: Signal, workspace: Workspace)
    - Logic:
      1. Ambil config alerts workspace: confidence_threshold, 
         slack_webhook_url, notification_email dari tabel alert_configs 
         (atau kolom di workspaces — sesuaikan dengan struktur DB yang ada)
      2. Cek apakah signal.confidence_score >= threshold
      3. Jika ya, kirim ke channel yang aktif:
         → Email jika notification_email ada
         → Slack jika slack_webhook_url ada

4b. Implementasi email via Resend (src/lib/email.ts — buat baru)
    - Setup Resend client dengan RESEND_API_KEY
    - Buat fungsi sendSignalAlertEmail(to: string, signal: Signal)
    - Template email:
      Subject: "🎯 New High-Intent Signal: {signal.title}"
      Body HTML (inline, tidak perlu file terpisah):
      - Judul sinyal
      - Platform (Reddit/HN) + subreddit/thread
      - Confidence score + intent level (High/Medium/Low)
      - AI summary
      - Suggested opener
      - Link "View Signal" → {app_url}/dashboard?signal={signal.id}
      - Link "Push to HubSpot" → tombol CTA
    - Tambahkan RESEND_API_KEY dan RESEND_FROM_EMAIL ke .env.example

4c. Implementasi Slack webhook (sudah ada URL di DB, tinggal kirim)
    - Buat fungsi sendSlackAlert(webhookUrl: string, signal: Signal)
    - Payload Slack Block Kit:

        "blocks": [
          { "type": "header", "text": "🎯 New High-Intent Signal" },
          { "type": "section", fields: [Platform, Confidence, Intent] },
          { "type": "section", "text": AI summary },
          { "type": "actions", elements: [
            "View Signal" button,
            "Push to HubSpot" button
          ]}
        ]
      }

4d. Integrasi ke ingest pipeline di src/lib/ingest.ts
    - Setelah berhasil menyimpan signal baru ke DB (langkah 1c):
      → Panggil sendAlertNotification(newSignal, workspace)
      → Wrap dalam try-catch — jika notifikasi gagal, 
        JANGAN gagalkan proses ingest utama (log error saja)

4e. Buat API route src/app/api/alerts/test/route.ts
    - Method: POST, Auth: user harus login
    - Kirim test notification ke config yang tersimpan
