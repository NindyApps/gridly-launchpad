FILE: supabase/migrations/add_salesforce_columns.sql

Isi migrasi:
  ALTER TABLE workspaces
    ADD COLUMN IF NOT EXISTS sf_access_token_enc       TEXT,
    ADD COLUMN IF NOT EXISTS sf_refresh_token_enc      TEXT,
    ADD COLUMN IF NOT EXISTS sf_instance_url           TEXT,
    ADD COLUMN IF NOT EXISTS sf_token_expires_at       TIMESTAMPTZ,
    ADD COLUMN IF NOT EXISTS sf_connected_at           TIMESTAMPTZ,
    ADD COLUMN IF NOT EXISTS sf_inject_object          TEXT 
      DEFAULT 'Task' 
      CHECK (sf_inject_object IN ('Task', 'Lead', 'Contact', 'Opportunity'));

Lalu jalankan: supabase db push

Penjelasan kolom:
- sf_access_token_enc   : Access token OAuth, dienkripsi AES sama 
                          seperti HubSpot
- sf_refresh_token_enc  : Refresh token OAuth (Salesforce tidak expire 
                          kecuali dicabut manual)
- sf_instance_url       : URL unik per org Salesforce, 
                          misal https://yourcompany.my.salesforce.com
                          WAJIB disimpan — semua API call pakai URL ini
- sf_token_expires_at   : Waktu kadaluarsa access token
- sf_connected_at       : Kapan workspace pertama kali connect
- sf_inject_object      : User pilih mau inject sinyal sebagai apa:
                          Task / Lead / Contact / Opportunity

LANGKAH 6A — Salesforce OAuth Flow
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Tujuan: User bisa klik "Connect Salesforce" → authorize di Salesforce 
→ kembali ke app dengan token tersimpan.

─────────────────────────────────────
6A-1. Buat src/lib/salesforce.ts
─────────────────────────────────────

Fungsi yang harus ada:

a) getSalesforceAuthUrl(state: string): string
   - Buat URL ke Salesforce OAuth 2.0 authorization endpoint
   - URL: https://login.salesforce.com/services/oauth2/authorize
   - Query params:
     response_type: 'code'
     client_id: process.env.SALESFORCE_CLIENT_ID
     redirect_uri: process.env.SALESFORCE_REDIRECT_URI
     scope: 'api refresh_token offline_access'
     state: state (random string untuk CSRF protection)
   - Return URL string lengkap

b) exchangeSalesforceCode(code: string): Promise<SalesforceTokenResponse>
   - POST ke https://login.salesforce.com/services/oauth2/token
   - Body (x-www-form-urlencoded):
     grant_type: 'authorization_code'
     client_id: SALESFORCE_CLIENT_ID
     client_secret: SALESFORCE_CLIENT_SECRET
     redirect_uri: SALESFORCE_REDIRECT_URI
     code: code
   - Return: { access_token, refresh_token, instance_url, 
               token_type, issued_at, expires_in? }

c) refreshSalesforceToken(workspaceId: string): Promise<void>
   - Ambil sf_refresh_token_enc dari DB, dekripsi
   - POST ke https://login.salesforce.com/services/oauth2/token
   - Body: grant_type: 'refresh_token', client_id, 
           client_secret, refresh_token
   - Enkripsi access_token baru
   - Update DB: sf_access_token_enc, sf_token_expires_at
   - CATATAN: Salesforce refresh token tidak pernah expired kecuali 
     di-revoke manual, jadi tidak perlu refresh refresh_token

d) getValidSalesforceToken(workspaceId: string): 
   Promise<{ accessToken: string, instanceUrl: string }>
   - Ambil sf_token_expires_at dari DB
   - Jika expires dalam < 5 menit → panggil refreshSalesforceToken()
   - Dekripsi dan return access token + instance_url
   - Throw error dengan pesan jelas jika workspace belum connect 
     Salesforce (sf_access_token_enc IS NULL)

e) revokeSalesforceToken(workspaceId: string): Promise<void>
   - POST ke {instance_url}/services/oauth2/revoke
   - Body: token={access_token}
   - Lalu clear semua kolom sf_* di DB (set NULL)

Tambahkan TypeScript interface:
  interface SalesforceTokenResponse {
    access_token: string
    refresh_token: string
    instance_url: string
    token_type: string
    issued_at: string
  }

Enkripsi/dekripsi: gunakan fungsi yang SAMA dengan yang sudah dipakai 
di HubSpot (cek src/lib/hubspot.ts atau src/lib/crypto.ts untuk 
memastikan konsisten)

─────────────────────────────────────
6A-2. Buat src/app/api/crm/salesforce/connect/route.ts
─────────────────────────────────────

- Method: GET
- Auth: user harus login (cek session Supabase)
- Logic:
  1. Generate random state string (crypto.randomUUID())
  2. Simpan state ke Supabase cookie atau session untuk verifikasi 
     di callback (cegah CSRF)
  3. Redirect ke getSalesforceAuthUrl(state)
- Tidak return JSON — langsung redirect

─────────────────────────────────────
6A-3. Buat src/app/api/crm/salesforce/callback/route.ts
─────────────────────────────────────

- Method: GET
- Query params yang diterima: code, state, error (optional)
- Logic:
  1. Jika ada query param error → redirect ke 
     /settings/crm?sf_error=access_denied
  2. Verifikasi state cocok dengan yang disimpan di cookie/session
  3. Panggil exchangeSalesforceCode(code)
  4. Enkripsi access_token dan refresh_token
  5. Hitung expires_at: 
     new Date(parseInt(tokenResponse.issued_at) + 
     (tokenResponse.expires_in ?? 7200) * 1000)
     CATATAN: issued_at dari Salesforce dalam milliseconds (bukan detik)
  6. Update tabel workspaces:
     sf_access_token_enc = encryptedAccessToken
     sf_refresh_token_enc = encryptedRefreshToken
     sf_instance_url = tokenResponse.instance_url
     sf_token_expires_at = expiresAt
     sf_connected_at = new Date()
  7. Redirect ke /settings/crm?sf_connected=true

─────────────────────────────────────
6A-4. Buat src/app/api/crm/salesforce/disconnect/route.ts
─────────────────────────────────────

- Method: POST
- Auth: user harus login
- Logic:
  1. Panggil revokeSalesforceToken(workspaceId) — handle error 
     dengan try-catch (jika revoke gagal di Salesforce, tetap 
     lanjutkan clear DB)
  2. Clear semua kolom sf_* di DB
  3. Return { success: true }

6B-1. Tambahkan fungsi inject ke src/lib/salesforce.ts
─────────────────────────────────────

Tambahkan fungsi-fungsi berikut:

a) injectAsTask(accessToken, instanceUrl, signal): Promise<string>
   - POST ke {instanceUrl}/services/data/v59.0/sobjects/Task/
   - Headers: Authorization: Bearer {accessToken}, 
              Content-Type: application/json
   - Body:
     {
       Subject: `[OCTOPILOT] ${signal.title}`,
       Description: `Platform: ${signal.platform}\n` +
                    `Confidence: ${signal.confidence_score}%\n` +
                    `Intent: ${signal.intent_level}\n\n` +
                    `AI Summary: ${signal.ai_summary}\n\n` +
                    `Suggested Opener: ${signal.suggested_opener}\n\n` +
                    `Original Post: ${signal.post_url}`,
       Status: 'Not Started',
       Priority: signal.intent_level === 'high' ? 'High' : 'Normal',
       ActivityDate: new Date(Date.now() + 86400000)
                     .toISOString().split('T')[0]  // tomorrow
     }
   - Return: Salesforce record ID (response.id)

b) injectAsLead(accessToken, instanceUrl, signal): Promise<string>
   - POST ke {instanceUrl}/services/data/v59.0/sobjects/Lead/
   - Body:
     {
       LastName: signal.author ?? 'Unknown (Reddit/HN)',
       Company: 'Unknown — via OCTOPILOT Signal',
       LeadSource: 'Web',
       Description: (same format as Task description above),
       Rating: signal.intent_level === 'high' ? 'Hot' : 'Warm',
       Website: signal.post_url
     }
   - Return: Salesforce record ID

c) injectAsContact(accessToken, instanceUrl, signal): Promise<string>
   - POST ke {instanceUrl}/services/data/v59.0/sobjects/Contact/
   - Body:
     {
       LastName: signal.author ?? 'Unknown (Reddit/HN)',
       Description: (same format),
       LeadSource: 'Web'
     }
   - Return: Salesforce record ID

d) injectAsOpportunity(accessToken, instanceUrl, signal): Promise<string>
   - POST ke {instanceUrl}/services/data/v59.0/sobjects/Opportunity/
   - Body:
     {
       Name: `[OCTOPILOT] ${signal.title}`,
       StageName: 'Prospecting',
       CloseDate: new Date(Date.now() + 30 * 86400000)
                  .toISOString().split('T')[0],  // 30 days from now
       Description: (same format),
       LeadSource: 'Web'
     }
   - Return: Salesforce record ID

e) injectSignalToSalesforce(
     workspaceId: string, 
     signal: Signal, 
     objectType?: 'Task'|'Lead'|'Contact'|'Opportunity'
   ): Promise<{ recordId: string, objectType: string, recordUrl: string }>
   - Fungsi utama yang dipanggil dari API route
   - Ambil { accessToken, instanceUrl } dari getValidSalesforceToken()
   - Ambil objectType dari parameter ATAU dari workspace.sf_inject_object
   - Panggil fungsi inject yang sesuai berdasarkan objectType
   - Susun recordUrl: 
     {instanceUrl}/lightning/r/{objectType}/{recordId}/view
   - Return { recordId, objectType, recordUrl }

─────────────────────────────────────
6B-2. Buat src/app/api/crm/salesforce/inject/route.ts
─────────────────────────────────────

- Method: POST
- Auth: user harus login
- Body: { signalId: string, objectType?: string }
- Logic:
  1. Ambil signal dari tabel intent_signals by signalId
     (pastikan signal milik workspace user ini)
  2. Panggil injectSignalToSalesforce(workspaceId, signal, objectType)
  3. Update tabel intent_signals:
     sf_injected_at = new Date()
     sf_record_id = result.recordId
     sf_record_url = result.recordUrl
  4. Return { 
       success: true, 
       recordId: result.recordId,
       recordUrl: result.recordUrl,
       objectType: result.objectType
     }
- Error handling:
  → 400 jika signalId tidak ada di body
  → 404 jika signal tidak ditemukan atau bukan milik workspace ini
  → 422 jika Salesforce belum di-connect (sf_access_token_enc NULL)
  → 502 jika Salesforce API error (forward pesan error dari SF)
